# 视觉风格定义 v2.0 — 活点笔记本

> **一本会动的活点笔记本。**
>
> 表面是一页严肃的笔记：像素字、网格排布、蓝色标题、灰底纸张。
> 凑近了看，墨迹未干，手绘的边框在微微呼吸，下划线在缓慢扭动，
> 高亮色块像荧光笔刚涂上去还没干透。
> 偶尔某一行正经文字里，冒出一句不该出现在这里的批注——
> 像这本笔记本的主人写累了，用爪子划拉了一句吐槽。
>
> 这是**正经与不正经的共生**，是活点地图的数码后代。

---

## 一、核心二元性

活点笔记本的美学建立在两组对立的精确平衡上：

```
表面正经 (The Grid)              底下不正经 (The Scribble)
─────────────────────────        ─────────────────────────
像素等宽字体，横平竖直             手绘线条在微微抖动
严格的 Grid 布局                  边框不闭合、不笔直
蓝色标题层级，严谨有序             高亮色块随机偏移
浅灰底纸感                       墨迹未干，像刚画上去
报纸/笔记般的栏目秩序              线条会动——画出自己、呼吸、扭动
```

**黄金比例**：90% 正经 + 10% 不正经。

- 正经占绝对主导——这是"笔记本"，不是涂鸦本。
- 不正经只出现在细节里（手绘的抖动、偶尔的动效、突然的批注）。
- 不正经不能掩盖正经的结构感——先让人相信这是笔记，再发现它是活的。

## 二、正经面 — 骨架层

这是笔记本的"常态"，是它站立起来的结构。

### 2.1 色彩

#### 蓝色标题系（越深越重要）

```
h1    #001BB7  ████████  最深蓝 — 页面大标题
h2    #0046FF  ████████  中蓝   — 章节标题
h3/h4 #3B82F6  ████████  浅蓝   — 小节标题
```

#### 装饰色（仅二色，点状使用）

```
橙色  #FF8040  ████████  下划线、重点线（线）
黄色  #F7CE46  ████████  高亮标记背景、边框（面）
```

#### 基底色

```
底纸  #E9E9E9  ████████  页面背景（主）
卡片底 #f4f4f4  ████████  卡片/输入框背景（比底纸稍亮）
正文  #1f2937  ████████  段落文字
深底  #111827  ████████  代码块背景
链接  #2563eb  ████████  超链接
```

#### 用色铁律

装饰色只用橙 + 黄，标题只用蓝系，正文只用深灰。

| ✅ 允许 | ❌ 禁止 |
|---|---|
| 蓝色做标题 | 蓝色做正文 |
| 橙色画线 | 橙色做大面积背景 |
| 黄色做高亮和边框 | 黄色做标题 |
| 正文 `#1f2937` | 正文用装饰色 |

### 2.2 字体

**像素等宽字体**。每个字像点阵打印机打出来的，横平竖直，字间距均等。这是"正经面"最重要的锚点——只要字体对，骨架就稳了。

网页实现用 `Fusion Pixel 12px Monospaced`，其他媒介用任意像素等宽字体替代。

#### 字号层级

```
h1    最大最重     页面主标题
h2    次大次重     章节标题
h3    中等粗细     小节标题
h4    较小较细     细节标题
p     正常大小     正文 (统一字号，靠间距分段落)
```

#### 排版习惯

- 段落间空行分隔，不用首行缩进
- 中英文/数字之间留半角空格
- 大段文字拆为短段落（每段 2-5 行）
- 像素字体天然等宽，表格/数据可自然对齐

### 2.3 布局

```
+----------+-------------------------+----------+
|          |        header           |          |
| leftbar  |     (标题区居中)          | rightbar |
| (导航)   +-------------------------+ (信息)   |
|          |                         |          |
|  • 链接  |     正文内容区            |  状态/    |
|  • 链接  |     (正常流)            |  参考     |
|  • 链接  |                         |          |
|          |                         |          |
+----------+-------------------------+----------+
|              footer                           |
+-----------------------------------------------+
```

- 首页：三栏 Grid（左导航 / 中内容 / 右信息）
- 子页面：单列居中，最大宽度约 800px

```
+-----------------------------------------------+
|                   header                       |
+-----------------------------------------------+
|                                               |
|                正文内容区                       |
|             (max-width: 800px, 居中)            |
|                                               |
+-----------------------------------------------+
|                   footer                       |
+-----------------------------------------------+
```

- 留白慷慨，内容不拥挤
- 固定元素：底部 footer，角落可放置彩蛋元素

## 三、不正经面 — 灵魂层

这是笔记本"活过来"的原因。所有效果的本质是：**有人刚刚用发抖的手，在这个页面上画了几笔**。

### 3.1 五个维度

| 维度 | 描述 |
|---|---|
| **墨迹未干** | 手绘线条有"刚画上去"的质感，不是印刷的 |
| **微微呼吸** | 边框/下划线有极缓慢的微动，像纸在呼吸 |
| **踩一脚浮现** | 元素可以延迟出现/滚动触发，像脚印一个个踩出来 |
| **不完美但自信** | 线是歪的，但歪得很笃定；不是手残，是风格 |
| **偶尔的批注** | 正经文字里突然出现一句不该在这的话 |

## 四、Sketchy 三件套

这是风格最具辨识度的视觉语言。三件套不是静态装饰——它们**会动**。

### 4.1 手绘边框 (sketchy-box)

```
┌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌┐
╎                                    ╎
╎   用略微发抖的手画出的矩形框          ╎
╎   线不直、角不尖、有重叠的多层笔触     ╎
╎                                    ╎
╎   颜色: 黄色 #F7CE46                ╎
╎   抖动: 中等幅度                    ╎
╎                                    ╎
└╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌┘
```

**静态特征**：
- 每条边由 1-4 条重叠的贝塞尔曲线构成，彼此轻微偏移
- 控制点随机偏移，幅度由粗糙度参数控制
- 有起笔顿笔的端点感

**动效**：
- **draw-in（画出自己）**：页面加载时，边框从左上角开始沿路径"画出"，持续 0.6-1.2s。四条边不是同时出现，而是顺时针依次画出，像一只手在描框。
- **breath（呼吸）**：画完之后，边框的粗糙度在 3-5s 周期内做 ±0.5 的微小波动。不是整个框在缩放，而是控制点在做布朗运动式的慢漂移。
- **hover 响应**：鼠标悬停时粗糙度微微增加（+1），离开后恢复。像被碰了一下。

**变体**：

| 场景 | 颜色 | 粗细 | 粗糙度 |
|---|---|---|---|
| 正文卡片 | 黄色 | 细 (~1.5px) | 中 (3/10) |
| 弹窗 | 黄色 | 略粗 (~2px) | 低 (1/10) |
| 大框强调 | 蓝色/橙 | 粗 (~6px) | 高 (4/10) |

#### CSS 等效实现

Houdini Paint Worklet 的"粗糙度"参数在纯 CSS 中没有直接对应。以下是 CSS/SVG 方案的等效映射：

| 概念 | Houdini 行为 | CSS/SVG 等效 |
|---|---|---|
| 粗糙度 +1 | control point offset += 1 | 伪元素 `::before` 的 filter 从 `url(#sketchy-light)` 切换为 `url(#sketchy)` |
| 粗糙度 +2 | control point offset += 2 | `border-width` +1px，配合 `filter: url(#sketchy)` |
| 呼吸 (breath) | 控制点布朗运动式慢漂移 | 伪元素 `opacity` 在 55%↔75% 之间，4-5s 周期波动 |
| 画出自己 (draw-in) | 路径 stroke-dashoffset 动画 | `opacity 0→1` + `translateY(6px→0)`，0.6-0.8s，各卡片错开延迟 |
| hover 响应 | 粗糙度增 +1 | 伪元素 filter 切换 + `border-width` 微增 |

#### 关键原则：滤镜仅作用于边框层

SVG `feDisplacementMap` 会扭曲元素的**所有**渲染内容——包括内部文字。绝不可将滤镜直接施加到含文字元素上。正确做法：滤镜只加在伪元素 `::before` / `::after` 上，用作独立的边框层。

```css
/* ❌ 错误：文字也会扭曲 */
.sketchy-card {
  border: 2px solid var(--yellow);
  filter: url(#sketchy);   /* 文字一并抖动 */
}

/* ✅ 正确：滤镜仅作用于伪元素边框层，文字保持清晰 */
.sketchy-card {
  position: relative;
  /* 元素自身无 filter、无 border */
}
.sketchy-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border: 2px solid var(--yellow);
  filter: url(#sketchy);        /* 只抖边框 */
  pointer-events: none;
  z-index: 1;
}
```

### 4.2 手绘高亮 (sketchy-mark)

```
  这是一段正文中的一个 ▓▓▓▓▓▓▓▓▓▓▓▓▓  关键词
                       锯齿折线色块
                       像荧光笔涂的但不整齐
```

**静态特征**：
- 文字下方的折线/锯齿图案色块
- 边界不光滑，有 ±1-8px 的随机偏移
- 从元素顶部到底部来回锯齿
- 颜色：黄色 `#F7CE46`

**动效**：
- **pulse（脉冲）**：色块透明度在 85%-100% 之间以 2-3s 周期缓慢波动。像荧光墨水还没完全干，在不同光线下显色深浅不一。
- **stagger-reveal（逐字浮现）**：当高亮标记出现在列表或连续文本中时，可以依次延迟浮现（每个间隔 100-200ms），像一排脚印。

**使用限制**：只标单个词或短词组（1-4 字），不标整句。

### 4.3 手绘下划线 (sketchy-underline)

```
   这是一个需要强调的短句
   ~~~~~~~~~~~~~~~~~~~~~~~~~~~    多条平行但不重合的波浪线
    ~~~~~~~~~~~~~~~~~~~~~~~~~     彼此交错、微微抖动
     ~~~~~~~~~~~~~~~~~~~~~~       颜色: 橙色 #FF8040
```

**静态特征**：
- 2-4 条平行波浪线，控制点位置彼此不同
- 每条线是二次贝塞尔曲线，控制点随机偏移 ±15%
- 线宽随机变化（粗细不匀）
- 终端微微上翘，像手写时的挑笔习惯

**动效**：
- **wiggle（微扭）**：贝塞尔控制点以 4-6s 周期做 ±3px 的慢漂移。多条线之间相位不同（各差 0.5-1s），视觉上形成"波浪追逐"——第一条线扭到波峰时，第二条线刚好在波谷。
- **draw-in（画出）**：加载时从左到右依次画出每条线，每条延迟 0.2s。像手从左到右划了 3 道。

#### CSS 实现限制

CSS `text-decoration` 仅支持单条线。2-4 条相位交错的波浪线无法用纯 CSS 实现。

**兜底方案**：使用 `text-decoration: underline wavy #FF8040`（单线），多线完整效果需 `<canvas>` 或内联 SVG 逐线绘制。

## 五、动效语汇

### 5.1 动效列表

| 动效 | 描述 | 周期 | 适用元素 |
|---|---|---|---|
| **draw-in** | 沿路径"画出自己" | 0.6-1.2s | 边框、下划线入场 |
| **breath** | 控制点慢漂移 | 3-5s | 边框持续性微动 |
| **wiggle** | 贝塞尔控制点扭动 | 4-6s | 下划线持续性微动 |
| **pulse** | 透明度缓慢波动 | 2-3s | 高亮标记持续性微动 |
| **stagger** | 列表项依次出现 | 每项 100-200ms | 任务列表、链接列表入场 |
| **reveal** | 从透明渐显 | 0.5-1s | 隐藏内容、彩蛋浮现 |
| **idle** | 偶尔的微小动作 | 间隔 5-15s | 角落彩蛋元素 |

### 5.2 动效原则

- **慢**——所有周期 2s 以上。快的动效破坏"纸张"感。
- **微**——偏移量在几个像素内。动效是让线条"呼吸"，不是让它们跳舞。
- **懒**——默认状态下动效极微，甚至可以暂停。hover 或滚动到视口时再激活。
- **可降级**——无动效时风格依然成立。动效是加分项，不是必需品。
- **不叠加**——同一区域内不超过两种动效同时运行。正经面是主角。

### 5.3 动效降级策略

```
网页:    CSS Houdini Paint Worklet (最佳) → CSS animation (备用) → 静态 (底线)
PPT:     极简 GIF / 无动效 (静态优先)
图片:    无动效，靠线条本身的抖动姿态暗示动态
纯文本:  无动效
```

### 5.4 降级代码片段

```css
/* 尊重用户系统偏好：关闭所有动画 */
@media (prefers-reduced-motion: reduce) {
  .sketchy-card::before,
  .sketchy-card::after,
  .sketchy-btn::before,
  .sketchy-mark,
  .settings-panel::before {
    animation: none !important;
  }
}

/* 不支持 Houdini 时回退到 SVG filter */
@supports not (background: paint(id)) {
  .sketchy-card::before {
    filter: url(#sketchy);
  }
}
```

## 六、交互元素 — 手写的暗示

### 6.1 核心理念

在活点笔记本里，**"可交互"不应该靠按钮的立体感或高饱和色来喊叫**。
取而代之的是一条简约的规则：

> 像素印刷体 = 只读内容。手绘痕迹 = 可以碰。

当用户看到一个 sketchy-box 包裹的区域，或一条 sketchy-underline 下的文字——这种"有人刚画过"的质感本身就暗示了可交互。
hover 和 focus 时，手绘痕迹加深，像用手指点了点那笔还没干的墨水。

```
默认态                     hover/focus                 active/点击
──────────────────────────────────────────────────────────────
边框: 细、淡黄             边框: 粗糙度+1              边框: 线变粗、颜色变橙
下划线: 静态/微扭          下划线: wiggle 加速          下划线: 多条线同时浮现
高亮: 静态脉冲             高亮: 脉冲加速、锯齿更密      高亮: 短暂闪亮
```

### 6.2 按钮

按钮是手绘笔记本里"被圈出来的动作"。不是圆角矩形，是被某人用笔框了一下，旁边可能还有潦草的标注。

**默认态**：
- sketchy-box 包裹，`--rough-stroke: #F7CE46` (黄)
- 内部文字像素等宽，颜色 `#1f2937`
- 粗糙度 2-3，线的抖动暗示"这是活的，可以按"
- 无背景色，无阴影，无圆角

**hover**：
- 粗糙度 +1，边框线看起来"刚被补了一笔"
- 内部文字颜色不变（保持可读）
- 如果按钮是主要操作（primary），边框色可从黄渐变到橙

**active（按下）**：
- 边框线变粗 +1px，粗糙度 +2
- 像一个手指按在笔画上，墨水微微洇开
- 持续 0.15-0.2s 后恢复

**变体**：

| 类型 | 边框色 | 描述 |
|---|---|---|
| 默认按钮 | 黄 `#F7CE46` | 日常操作 |
| 主要按钮 | 橙 `#FF8040` | 强调操作（仅一个） |
| 危险按钮 | 深红提示 | 删除等，hover 时下划线变红 |
| 幽灵按钮 | 无边框 | hover 时出现 sketchy-underline |

### 6.3 链接

正文中的链接是"被某人划过线的词"。

**默认态**：
- 文字色 `#2563eb`，底部 1 条 sketchy-underline (`--lineNumber: 1`)
- 下划线静态（不扭动），颜色橙 `#FF8040`
- 区别于正文装饰下划线（3 条、会扭）——链接的线少而安静

**hover**：
- 下划线 `--lineNumber` 从 1 变为 2-3，wiggle 动效激活
- 粗糙度微增，像又补了两道
- 文字色不变

**visited（已访问）**：
- 下划线颜色变淡（橙 → 浅橙），线条数保持 1
- 暗示"这行被划过，但旧了"

### 6.4 输入框

输入框是笔记本上"留的空"。不是白色方框，是虚线或淡手绘线框住的空白区域。

**默认态**：
- sketchy-box 包裹，`--rough-stroke: #d1d5db` (灰)
- 粗糙度 1-2，比内容区边框更淡
- 内部底色 `#f4f4f4`（比页面底稍亮），placeholder 文字灰色
- 像用铅笔画的线，等你去填

**focus**：
- 边框色从灰 → 黄 `#F7CE46`
- 粗糙度 +2，呼吸动效激活
- 光标使用像素等宽的光标形状（blink 速度慢 30%）

**filled（有内容）**：
- 边框色保持黄，但粗糙度回到默认
- 文字 `#1f2937`，像已经写在里面了

#### CSS 实现限制

`<input>` 和 `<textarea>` 是替换元素，不支持 `::before` / `::after` 伪元素。因此 sketchy-box 的滤镜边框无法直接作用于输入框本身。建议对输入框使用普通 `border` + 颜色过渡，将 sketchy 手绘效果保留给外层卡片容器。

### 6.5 复选框 / 勾选

复选框不应该是浏览器原生的方形勾选框，而是一个**手绘的方框 + 手绘的勾**。

**未勾选**：
- 一个 sketchy-box 小方框（~18px），粗糙度 3
- 线色黄 `#F7CE46`，细线
- 旁边的任务描述是正常正文
- 方框微微呼吸（周期 4s）

**hover**：
- 方框粗糙度 +1
- 线色偏橙，像笔尖悬在纸上

**勾选瞬间**：
- 框内出现一个手绘的"✓"，不是字体，是画出来的路径
- 勾的 draw-in 动画：0.3s，从左上到右下
- 勾的线色橙 `#FF8040`，线宽略粗于边框
- 勾好后，边框线色变灰，任务描述变淡 + 删除线
- 删除线也是一条 sketchy-underline（--lineNumber: 1, 橙色）

### 6.6 开关 / Toggle

开关是一个**手绘的滑块槽 + 手绘的小圆**。

**关状态**：
- 一条短粗的手绘线段（~36px 长），灰 `#d1d5db`
- 线段左端有一个空心圆（手绘的圈，未填充）
- 像用铅笔画了个半成品的开关

**开状态**：
- 线段色变黄 `#F7CE46`
- 圆从左侧滑到右侧（过渡 0.3s，用 ease-out，但要有一点"弹过再回"的微妙感）
- 圆从空心变为实心（手绘填充点）

**hover**：
- 线段粗糙度 +1
- 圆微微变大（+2px 半径）

### 6.7 可折叠区域 (toggle-section)

折叠区块的提示是一个**手绘的三角形** `▶`。

**折叠态**：
- `▶` 由 3 条手绘线组成，不是字体字符
- 颜色灰 `#d1d5db`，粗糙度 2
- 右侧标题文字正常

**hover**：
- 三角形线色变橙
- 粗糙度 +1

**展开态**：
- 三角形旋转 90° 指向下方（过渡 0.25s）
- 旋转后三角形线色保持黄 `#F7CE46`
- 内容区从折叠态展开（高度过渡，内容区顶部画一条淡 sketchy-underline 作为分隔）

### 6.8 通用交互规则

```
所有可交互元素共享以下行为模式：

1. 默认态融入 — 不喧哗，手绘质感本身是暗示
2. hover 时手绘加深 — 粗糙度 +1、线条数 +1、或动效加速
3. active 时线条变粗 — 像手指按在墨迹上
4. focus 时边框高亮 — 键盘导航也能触发手绘反应
5. 过渡要慢 — 所有 hover/active 过渡 ≥ 0.2s
6. 不使用系统原生控件 — 复选框、开关、输入框全部手绘替代
```

### 6.9 交互元素禁忌

| ❌ 禁止 | ✅ 替代 |
|---|---|
| 浏览器原生 checkbox/radio | 手绘方框 + 手绘勾 |
| 3D 按钮（阴影、渐变） | sketchy-box 按钮 |
| 默认蓝色文字链接 | sketchy-underline 链接 |
| 默认 input 白底方框 | sketchy-box 输入框 |
| hover 时颜色跳变 | hover 时粗糙度渐变 |
| 过渡时间 < 0.15s | 所有交互过渡 ≥ 0.2s |
| 系统默认 focus 轮廓 | sketchy-box 边框色变化 |

## 七、文案语调 — 正经壳 × 不正经馅

### 7.1 核心原则

**90% 的文字是正经的**：信息清晰、结构完整、像一篇真正的笔记或报纸。
**10% 的文字突然不正经**：一句批注、一句吐槽、一句"不该出现"的话。

这 10% 不能集中在某处——它应该**散落**在正经文字之间，像笔记本的主人在旁边用铅笔加了一句。

### 7.2 语调层级

| 层级 | 占全文比例 | 口吻 | 示例 |
|---|---|---|---|
| 结构层（标题/导航/标签） | ~15% | 完全正经 | "今日专注"、"任务列表"、"关于" |
| 正文层（段落/列表/说明） | ~75% | 正经为主 | 信息清晰、有条理、像素网格 |
| 批注层（偶尔的插话） | ~10% | 突然不正经 | 见下方示例 |

### 7.3 批注层示例

散落在正经文字之间的"不正经馅"：

```
"先把爪子搁键盘上。打字也好，刨土也好，动起来再说。"

"专注不是一种技能，是龙的呼吸。"

"龙在刨洞的时候从不看时间，但洞就是越刨越深。"

"这里什么内容都没有，只是毛坯窝。"

"无用的小知识：网站的设计风格是'奇怪的像素' + '奇怪的手绘'。"

"传送阵，起！"
```

### 7.4 语调节奏示例

```
## 今日专注                              ← 正经 h2
                                                                                
工欲善其事，必先利其器。                    ← 正经正文
先把爪子放上去再说。                       ← 突然不正经
打字也好，刨土也好。                       ← 继续不正经，一行就收
                                                                                
## 任务列表                              ← 继续正经结构
                                                                                
- [ ] 完成样式指南                         ← 正经任务
- [ ] 写两行代码                           ← 正经任务
- [ ] 刨个洞                              ← 混进去了
                                                                                
专注不是一种技能，是龙的呼吸。               ← 作为段落尾句收束，不正经但点题
```

### 7.5 语调禁忌

| ❌ 禁止 | ✅ 替代 |
|---|---|
| 全文不正经（变成搞笑文） | 90% 正经 + 10% 不正经 |
| 不正经集中在开头（一上来就破功） | 散落在各处，像彩蛋 |
| 不正经长篇大论（超过两行） | 一句就够，最多两句 |
| 商业/营销口吻（"点击立即体验"） | 笔记/日志口吻（"写完了，休息一下"） |
| 完全严肃毫无破绽 | 至少留一个位置可以"破功" |

## 八、龙 — 偶尔路过的彩蛋

龙**不是**品牌人格，不是自称，不是主角。主角是笔记本身。

龙是一个**偶尔出现的彩蛋**，作用是在正经的笔记里制造"刚才是不是有什么东西动了一下"的微妙感。

### 龙的出现方式

| 形式 | 频率 | 描述 |
|---|---|---|
| 意外插入的批注 | 偶尔 | 在正文中作为"不正经馅"出现（如那些例句） |
| 404 页面 | 仅 404 | 404 页面可以是龙最活跃的地方 |

### 龙不能做的事

- 不能出现在每个页面的 header
- 不能作为普遍的叙事者（"龙觉得……"）
- 不能占据 footer 的核心信息位置
- 三条例句是好东西，但散开用，不要集中

## 九、跨媒介适配

### 9.1 网页

| 元素 | 实现 |
|---|---|
| 手绘边框 | SVG `feTurbulence` + `feDisplacementMap` 滤镜，作用于伪元素 `::before` / `::after` 作为边框层 |
| 手绘边框动效 | CSS animation 控制伪元素 `opacity`（呼吸）、`border-width`（hover）、`opacity` + `translateY`（入场画出） |
| 手绘高亮 | CSS `linear-gradient` 多色块模拟锯齿边界，透明度随机跳变 |
| 手绘高亮动效 | CSS animation 控制 `opacity` 脉冲（2-3s 周期） |
| 手绘下划线 | `text-decoration: underline wavy #FF8040`（单线兜底）；多线完整效果需 `<canvas>` 或内联 SVG |
| 手绘下划线动效 | CSS animation 控制位移微摆；或 canvas 逐帧更新贝塞尔控制点 |
| 像素字体 | `@font-face` 加载本地 woff2（优先），CDN 备选，`Courier New` / `monospace` 兜底 |
| 底色 | `background-color: #E9E9E9` |
| 动效降级 | `@media (prefers-reduced-motion: reduce)` 关闭动画；不支持 SVG filter 时回退到纯色边框（静态底线） |

> **注**：CSS Houdini Paint Worklet（`roughBox` / `sketchyBackground` 等）在支持它的浏览器中是更优解，但目前仅 Chrome 支持，不应作为唯一实现路径。上述 SVG filter + CSS animation 方案覆盖全部主流浏览器。

### 9.2 PPT / Keynote

| 元素 | 实现 |
|---|---|
| 手绘边框 | RoughJS 手绘矩形 / 自由曲线手动描边 |
| 手绘高亮 | 半透明黄色多边形的锯齿路径 |
| 手绘下划线 | 多条橙色贝塞尔曲线 |
| 像素字体 | 安装像素字体到系统（如 Zpix），或截图嵌入 |
| 底色 | 幻灯片背景 `#E9E9E9` |
| 动效 | GIF / 极简路径动画 / 或无动效，静态优先 |

### 9.3 图片 / 海报

| 元素 | 实现 |
|---|---|
| 手绘边框 | 粗糙圆头笔刷徒手描框 |
| 手绘高亮 | 半透明黄笔刷涂抹 |
| 手绘下划线 | 橙色彩铅画 3 道不均匀线 |
| 像素字体 | 字体转曲后使用 |
| 底色 | 纸张纹理底，颜色偏向 `#E9E9E9` |
| 动效 | 无。靠线条本身的抖动姿态暗示动态 |
| 彩蛋 | 可在角落手写一句小字批注 |

### 9.4 纯文字 / Markdown

| 元素 | 替代方案 |
|---|---|
| 手绘边框 | ASCII 框线 `┌─┐` `│ │` `└─┘` |
| 手绘高亮 | Markdown 加粗 `**关键词**` |
| 手绘下划线 | 分隔线 `---` 暗示 |
| 像素字体 | 代码块内等宽字体 |
| 底色 | 依赖渲染环境 |
| 动效 | 无 |
| 彩蛋 | `<!-- 注释里的批注 -->` 或脚注形式 |

## 十、禁忌

以下做法会破坏这个风格：

| ❌ 禁止 | 原因 |
|---|---|
| 圆角边框 | 手绘边框自带粗糙感，圆角过于精致 |
| CSS 渐变 | 手绘靠线条抖动制造质感，不是颜色渐变 |
| 阴影做层次 | 手绘的厚度来自重叠笔触，不是投影 |
| 装饰色做大面积背景 | 橙黄是点缀，铺满刺眼 |
| 标题用非蓝色 | 蓝色系层级是重要视觉秩序 |
| 正文用装饰色 | 正文保持 `#1f2937` |
| 字体混用 | 像素等宽是基底，不要混入其他字体 |
| 太整齐的线 | 手绘的"缺陷"是故意的 |
| 动效太快 | 所有动效周期 ≥ 2s |
| 动效太多 | 同一区域 ≤ 2 种动效同时运行 |
| 全文不正经 | 90% 正经是底线，否则变成搞笑页面 |
| 龙到处出现 | 龙是彩蛋，不是主角 |

## 十一、一句话概括

> 一本像素排印的活点笔记本——严肃地排版，随意地手绘，缓慢地呼吸，偶尔在正经文字里混进一句不该出现在这儿的批注。像有人在用爪子认真写东西，但爪子不太灵活，而且它觉得这样挺好的。
